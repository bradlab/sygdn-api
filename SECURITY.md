# Security Policy (English)

## Supported Versions

We actively maintain the latest major version of `sygdn-api`. Please ensure you are using the most recent release to benefit from security updates and fixes.

| Version       | Supported          |
| ------------- | ----------------- |
| Latest (main) | :white_check_mark: |
| Older         | :x:               |

## Reporting a Vulnerability

If you discover a security vulnerability in this project, **please do not open a public issue or pull request**.  
Instead, report it confidentially by contacting us at: [contact@bradlab.dev]

Please include as much detail as possible in your report, including:

- Steps to reproduce the issue
- The impact of the vulnerability
- Any suggestions for remediation (if possible)
- Your contact details for follow-up (optional)

We will acknowledge receipt of your report within 72 hours and keep you informed of our progress in resolving the issue.

## Security Update Process

- We investigate all security reports promptly and thoroughly.
- If a vulnerability is confirmed, we will work to release a fix as soon as possible.
- After a fix is released, we will publicly disclose the vulnerability, crediting the reporter if desired.

## Best Practices

- Always keep your dependencies up-to-date.
- Do not share sensitive project credentials in issues, pull requests, or other public channels.
- Use environment variables for secrets and configuration.

Thank you for helping us keep sygdn-api secure!

# Politique de sécurité

## Versions supportées

Nous maintenons activement la dernière version majeure de `sygdn-api`. Veuillez utiliser la version la plus récente pour bénéficier des mises à jour et correctifs de sécurité.

| Version        | Supportée           |
| -------------- | ------------------ |
| Dernière (main)| :white_check_mark: |
| Anciennes      | :x:                |

## Signaler une vulnérabilité

Si vous découvrez une vulnérabilité de sécurité dans ce projet, **ne créez pas de ticket public ou de pull request**.  
Veuillez la signaler de manière confidentielle en contactant : [contact@bradlab.dev]

Merci d’inclure autant de détails que possible dans votre rapport, notamment :

- Étapes pour reproduire la faille
- Impact de la vulnérabilité
- Suggestions de remédiation (si possible)
- Vos coordonnées pour un suivi (optionnel)

Nous accuserons réception de votre rapport sous 72h et vous tiendrons informé de l’avancement de la résolution.

## Processus de gestion des incidents de sécurité

- Tous les signalements sont analysés rapidement et de manière approfondie.
- Si une vulnérabilité est confirmée, nous publierons un correctif au plus vite.
- Après publication du correctif, la faille sera rendue publique et le découvreur pourra être crédité s’il le souhaite.

## Bonnes pratiques

- Gardez toujours vos dépendances à jour.
- Ne partagez jamais d’identifiants sensibles (secrets, mots de passe, tokens) dans les issues, pull requests ou autres canaux publics.
- Utilisez les variables d’environnement pour les secrets et la configuration.

Merci de contribuer à la sécurité de sygdn-api !
